## 1. 系统概述

本系统为 4 通道直流有刷电机驱动方案，基于 TI MSPM0G3507 微控制器，采用 DRV8870DDAR H 桥驱动芯片配合 S8050 NPN 三极管反相器实现单 PWM 锁相驱动（Locked Anti-Phase）控制。每个通道配备独立的电流采样电路（LMV321 运放 + 150mΩ 检流电阻），通过 ADC 反馈至 MCU 实现电流监控。

### 1.1 信号链路

```
MCU PWM ──┬──→ DRV8870 IN1 (直连)
          │
          └──→ S8050 反相器 ──→ DRV8870 IN2

DRV8870 OUT1/OUT2 ──→ 电机
DRV8870 ISEN ──→ R12(150mΩ) ──→ GND
                └──→ LMV321(×10) ──→ ADC ──→ MCU
```

IN1 与 IN2 始终互补：PWM 高电平时 IN1=1/IN2=0（正转），低电平时 IN1=0/IN2=1（反转）。50% 占空比时正反转时间各半，平均电压为零，电机停止并产生主动阻尼。

### 1.2 通道分配

| 通道 | DRV8870 | 运放 | PWM 定时器通道 | PWM 引脚 | 编码器 | ADC |
|------|---------|------|-------------|---------|--------|-----|
| A (左前) | U3 | U7 | TIMA0 CC0 | PB8 | TIMG7 | 待配置 |
| B (左后) | U5 | U10 | TIMA0 CC1 | PA22 | TIMA1 | 待配置 |
| C (右前) | U8 | U9 | TIMA0 CC2 | PA15 | TIMG6 | 待配置 |
| D (右后) | U11 | U12 | TIMA0 CC3 | PA17 | TIMG0 | 待配置 |

### 1.3 双驱动并存说明

项目中同时存在两套电机驱动代码：

| 驱动文件 | 芯片 | 控制方式 | 状态 |
|---------|------|---------|------|
| `bsp_motor.c/h` | TB6612FNG | sign-magnitude（PWM + IN1/IN2 GPIO） | 应用层当前使用 |
| `bsp_drv8870.c/h` | DRV8870 | 锁相驱动（单 PWM） | 代码正确但未被调用 |

硬件实际为 DRV8870 拓扑，应用层需从 `bsp_motor` 切换到 `bsp_drv8870`。两套驱动 API 形状一致，切换不涉及接口重构。

## 2. 硬件接口规范

### 2.1 DRV8870 引脚定义

DRV8870DDAR 采用 HSOP-8 封装（带散热焊盘），引脚定义依据 TI 数据手册 SLVSCY8B：

| 引脚号 | 引脚名 | 类型 | 功能 | 本项目连接 |
|-------|--------|------|------|-----------|
| 1 | GND | PWR | 逻辑地 | GND |
| 2 | IN2 | I | 逻辑输入 2，内部下拉 100kΩ | Mx_PWM2（S8050 反相输出） |
| 3 | IN1 | I | 逻辑输入 1，内部下拉 100kΩ | Mx_PWM1（MCU 直连） |
| 4 | VREF | I | 电流调节参考（0.3~5V） | 3V3 |
| 5 | VM | PWR | 电机电源（6.5~45V） | VIN_OUT |
| 6 | OUT1 | O | H 桥输出 1 | 电机端 A |
| 7 | ISEN | PWR | 电流检测 | R12(150mΩ) → GND |
| 8 | OUT2 | O | H 桥输出 2 | 电机端 B |
| PAD | — | — | 散热焊盘 | GND（大面积铜箔） |

### 2.2 S8050 反相器电路

每个通道使用 1 个 S8050 NPN 三极管构建共射极反相器：

| 元件 | 参数 | 功能 |
|------|------|------|
| Q1~Q4 | S8050 (0.5A) | NPN 三极管，共射极反相 |
| 基极限流电阻 | 1kΩ | 限制基极电流 |
| 基极下拉电阻 | 10kΩ | MCU 高阻态时确保三极管截止 |
| 集电极上拉电阻 | 1kΩ | 三极管截止时拉高输出 |

工作逻辑：输入高 → 三极管导通 → 输出低；输入低 → 三极管截止 → 上拉为高。

### 2.3 电流采样电路

| 元件 | 参数 | 功能 |
|------|------|------|
| R12/R18/R16/R28 | 150mΩ | 检流电阻（ISEN 与 GND 之间） |
| R4/R6/R14/R19 | 10kΩ | 运放反相输入电阻 |
| R5/R8/R15/R20 | 90kΩ | 运放反馈电阻 |
| U7/U10/U9/U12 | LMV321IDBVR | 单通道轨到轨运放 |
| C5/C6/C7/C9 | 100nF | 运放输入滤波 |

运放增益：Av = 1 + 90kΩ/10kΩ = 10 倍。

### 2.4 电气参数

#### DRV8870 关键参数（SLVSCY8B, TA=25°C）

| 参数 | 符号 | 最小 | 典型 | 最大 | 单位 |
|------|------|------|------|------|------|
| 电源电压 | VM | 6.5 | — | 45 | V |
| 峰值输出电流 | I_peak | 0 | — | 3.6 | A |
| 连续输出电流 | I_cont | 0 | — | 3.5 | A |
| 高侧导通电阻 | RDS(on)_HS | — | 307 | 360 | mΩ |
| 低侧导通电阻 | RDS(on)_LS | — | 258 | 320 | mΩ |
| 传播延迟 | tPD | — | 0.7 | 1.0 | μs |
| 死区时间 | tDEAD | — | 220 | — | ns |
| 内部下拉电阻 | RPD | — | 100 | — | kΩ |
| 输入高电平电流 | IIH | — | 33 | 100 | μA |
| 过流保护 | IOCP | 3.7 | 4.5 | 6.4 | A |
| 热关断 | TSD | 150 | 175 | — | °C |

#### 电流调节计算

```
I_TRIP = VREF / (AV × R_SENSE) = 3.3V / (10 × 0.15Ω) = 2.2A（典型）
```

AV 范围 9.4~10.4，因此 I_TRIP 范围 2.12~2.34A。

#### 电流采样量程

| 参数 | 计算值 | 说明 |
|------|--------|------|
| 运放增益 | 10 倍 | R5=90kΩ, R4=10kΩ |
| 满量程电流 | 2.2A | 3.3V / (10 × 0.15) |
| ADC 分辨率 | 0.54mA | 3.3V / (4096 × 10 × 0.15) |
| LMV321 实际最大输出 | ≈3.2V | 轨到轨余量 0.1V |
| 实际最大可测电流 | ≈2.13A | 3.2V / (10 × 0.15) |

运放饱和点（2.13A）低于硬件限流点（2.12~2.34A），存在约 0.07~0.21A 的测量盲区。建议将 R5 从 90kΩ 改为 82kΩ（增益 9.2 倍），按 LMV321 实际最大输出 3.2V 计算满量程为 3.2V / (9.2 × 0.15Ω) ≈ 2.32A，可覆盖硬件限流上限。

### 2.5 DRV8870 真值表

| IN1 | IN2 | OUT1 | OUT2 | 模式 |
|-----|-----|------|------|------|
| 0 | 0 | High-Z | High-Z | Coast（1ms 后休眠） |
| 0 | 1 | L | H | 反转 |
| 1 | 0 | H | L | 正转 |
| 1 | 1 | L | L | Brake（低侧慢衰减） |

锁相驱动下 IN1 和 IN2 始终互补，因此只会出现正转/反转两种状态交替，不会进入 Coast 或 Brake 模式。

### 3.2 HAL Timer 抽象

`hal_timer.c` 封装了 DL_Timer 接口，通过 `hal_timer_id_t` 枚举映射定时器实例：

```c
typedef enum {
    HAL_TIMER_PWM_MOTOR = 0,  // TIMA0 - 电机PWM (4通道)
    HAL_TIMER_CAPTURE_LF,     // TIMG7 - 左前编码器
    HAL_TIMER_CAPTURE_LB,     // TIMA1 - 左后编码器
    HAL_TIMER_CAPTURE_RF,     // TIMG6 - 右前编码器
    HAL_TIMER_CAPTURE_RB,     // TIMG0 - 右后编码器
    HAL_TIMER_SYS_TICK,       // TIMG8 - 系统节拍
    HAL_TIMER_COUNT
} hal_timer_id_t;
```

PWM 占空比通过 `hal_timer_set_pwm_duty()` 设置，内部调用 `DL_Timer_setCaptureCompareValue()`。所有定时器在 `SYSCFG_DL_init()` 中完成时钟/模式/周期配置，HAL 层仅提供运行时控制。

### 3.3 BSP DRV8870 驱动

`bsp_drv8870.c/h` 实现锁相驱动逻辑，核心 API：

| 函数 | 功能 |
|------|------|
| `bsp_drv8870_init()` | 初始化，启动 PWM 定时器，所有通道设为 50% duty |
| `bsp_drv8870_set_speed(motor, speed)` | 设置速度（正值正转，负值反转，0 停止） |
| `bsp_drv8870_stop(motor, mode)` | 停止电机（50% duty 主动阻尼） |
| `bsp_drv8870_stop_all()` | 停止所有电机 |
| `bsp_drv8870_get_duty_max()` | 获取占空比最大值 |
| `bsp_drv8870_percent_to_duty(percent)` | 百分比转 duty 值 |

配置结构体：

```c
typedef struct {
    uint32_t pwm_ch;           // PWM通道索引 (CC0~CC3)
    int8_t   dir_sign;         // 方向修正 (+1/-1, 0视为+1)
    int16_t  zero_duty_offset; // 零点偏移补偿 (需实测标定)
} bsp_drv8870_config_t;
```

## 4. 项目初始化指南

### 4.1 SysConfig 配置

SysConfig 生成 `ti_msp_dl_config.c/h`，配置所有外设引脚和时钟。以下为 DRV8870 相关配置要点。

#### 4.1.1 PWM 定时器（TIMA0）

| 配置项 | 当前值 | 说明 |
|--------|--------|------|
| 时钟源 | BUSCLK | 80MHz（= MCLK，SYSPLL 倍频后） |
| 分频比 | DIVIDE_4 | 80MHz / 4 = 20MHz |
| PWM 模式 | EDGE_ALIGN_UP | 边沿对齐向上计数 |
| Period (ARR) | 1000 | 20kHz (20MHz / 1000) |
| CC0~CC3 初始值 | **0** | **需改为 500**（见 §6.2 问题 3） |

> **注**：SysConfig 生成的注释 `20000000 Hz = 20000000 Hz / (4 * (0 + 1))` 存在已知数学错误（20M/4 ≠ 20M），实际 timer 时钟以 BUSCLK=80MHz、DIVIDE_4、prescale=0 推导为 20MHz。注意 ULPCLK（= MCLK/2 = 40MHz）与 BUSCLK 不同，TimerA 使用 BUSCLK 而非 ULPCLK。

PWM 引脚映射（`ti_msp_dl_config.h`）：

| 通道 | 引脚 | PINCM | 复用功能 |
|------|------|-------|---------|
| CC0 | PB8 | PINCM25 | IOMUX_PINCM25_PF_TIMA0_CCP0 |
| CC1 | PA22 | PINCM47 | IOMUX_PINCM47_PF_TIMA0_CCP1 |
| CC2 | PA15 | PINCM37 | IOMUX_PINCM37_PF_TIMA0_CCP2 |
| CC3 | PA17 | PINCM39 | IOMUX_PINCM39_PF_TIMA0_CCP3 |

#### 4.1.2 方向 GPIO（TB6612 用，DRV8870 不需要）

`ti_msp_dl_config.h` 中定义了 TB6612 的 IN1/IN2 引脚（MOTOR_AIN1~DIN2）。DRV8870 锁相驱动不需要这些引脚，切换后这些 GPIO 可释放用于其他功能。

#### 4.1.3 SysConfig 修改要点

切换到 DRV8870 驱动时，需要在 SysConfig 中修改：

1. **PWM CC 初始值**：所有 4 个 CC 的 `setCaptureCompareValue` 从 0 改为 500
2. **（可选）移除方向 GPIO**：MOTOR_AIN1~DIN2 的 GPIO 配置可删除
3. **（可选）增加 ADC 通道**：为 4 路电流采样添加 ADC 通道配置

### 4.2 HAL 初始化

HAL 层无需显式初始化函数，所有外设由 `SYSCFG_DL_init()` 完成硬件配置。HAL 层仅在运行时通过 `hal_timer_start()` 等函数控制外设启停。

`hal_timer.c` 中的定时器映射表：

```c
static const timer_instance_t s_timer_map[HAL_TIMER_COUNT] = {
    { (GPTIMER_Regs *)TIMA0, TIMA0_INT_IRQn, true  },  // PWM_MOTOR
    { (GPTIMER_Regs *)TIMG7, TIMG7_INT_IRQn, false },  // CAPTURE_LF
    { (GPTIMER_Regs *)TIMA1, TIMA1_INT_IRQn, false },  // CAPTURE_LB
    { (GPTIMER_Regs *)TIMG6, TIMG6_INT_IRQn, false },  // CAPTURE_RF
```

### 4.3 BSP DRV8870 初始化

`bsp_drv8870_init()` 执行以下步骤：

```
1. 检查参数合法性（cfg 非空、count 非零、pwm_period 非零）
2. 保存配置表、定时器实例、PWM 周期
3. 计算 50% 零速点: pwm_zero_point = pwm_period / 2
4. 启动 PWM 定时器: hal_timer_start()
5. 遍历所有通道，设置初始 duty = 50% + zero_offset
6. 标记初始化完成
```

调用方式：

```c
static const bsp_drv8870_config_t s_drv8870_cfg[BSP_DRV8870_COUNT] =
    PRJ_DRV8870_CONFIGS;

bsp_status_t ret = bsp_drv8870_init(
    s_drv8870_cfg,
    BSP_DRV8870_COUNT,
    PRJ_DRV8870_PWM_TIMER,    // HAL_TIMER_PWM_MOTOR
    PRJ_DRV8870_PWM_PERIOD    // 1000
);
```

`PRJ_DRV8870_CONFIGS` 在 `project_config.h` 中定义：

```c
#define PRJ_DRV8870_CONFIGS { \
    { 0U, 1, 0 },  // CC0, dir=+1, offset=0
    { 1U, 1, 0 },  // CC1, dir=+1, offset=0
    { 2U, 1, 0 },  // CC2, dir=+1, offset=0
    { 3U, 1, 0 },  // CC3, dir=+1, offset=0
}
```

### 4.5 开机安全

#### 4.5.1 开机暴走问题

MCU 复位期间 GPIO 处于高阻态，此时：

- DRV8870 IN1 被内部下拉电阻（100kΩ）拉低 → IN1 = 0
- S8050 基极被下拉电阻（10kΩ）拉低 → 三极管截止 → 集电极被上拉（1kΩ）→ IN2 = 1
- 结果：IN1=0, IN2=1 → 电机全速反转

4 个通道全部受影响。

#### 4.5.2 时间线

```
t0  上电               → GPIO 高阻态 → 电机全速反转
t1  MCU POR 释放        → 时钟启动 (~1-2ms) → 电机仍全速反转
t2  SYSCFG_DL_init()    → 定时器/GPIO 配置 → CC=0 → 电机仍全速反转
t3  FreeRTOS 启动       → bsp_drv8870_init() → CC=500 → 电机停止
```

t0 到 t3 的时间约 50~100ms，期间 4 个电机全速反转。

#### 4.5.3 缓解措施

| 措施 | 实施位置 | 效果 |
|------|---------|------|
| SysConfig CC 初始值改为 500 | `ti_msp_dl_config.c` | t2 时刻即停止，窗口缩短至 t0~t2 |
| 早期 GPIO 配置 | 启动汇编 `Reset_Handler` | 进一步缩短至 t0~t1（2~5ms） |
| P-MOSFET 控制上拉供电 | 硬件修改 | 彻底消除（需增加 4 个 P-MOSFET） |

仅修改 SysConfig CC 初始值即可将危险窗口从 50~100ms 缩短到约 2~5ms（POR + 时钟启动时间），这是不改硬件情况下最有效的软件缓解措施。

## 5. PWM 控制逻辑

### 5.1 锁相驱动原理

MCU 输出单路 PWM 信号，直连 DRV8870 IN1，同时经 S8050 反相后接入 IN2。IN1 和 IN2 始终互补：

| PWM 占空比 | IN1 | IN2 | 平均电压 | 电机行为 |
|-----------|-----|-----|---------|---------|
| 50% | 1/0 交替 | 0/1 交替 | 0V | 停止（主动阻尼） |
| >50% | 1 长 | 0 长 | >0V | 正转 |
| <50% | 0 长 | 1 长 | <0V | 反转 |
| 100% | 恒 1 | 恒 0 | +VM | 全速正转 |
| 0% | 恒 0 | 恒 1 | -VM | 全速反转 |

平均电压公式：V_avg = (2 × D - 1) × VM

### 5.2 Speed 到 Duty 映射

`bsp_drv8870.c` 中 `speed_to_duty()` 函数的核心公式：

```c
duty = PWM_PERIOD/2 + zero_duty_offset + clamp(speed × dir_sign, ±PWM_PERIOD/2)
```

以 PWM_PERIOD=1000 为例的完整映射：

| speed | dir_sign | clamped | duty | 占空比 | 行为 |
|-------|----------|---------|------|--------|------|
| +500 | +1 | +500 | 1000 | 100% | 全速正转 |
| +250 | +1 | +250 | 750 | 75% | 正转 50% |
| 0 | +1 | 0 | 500 | 50% | 停止 |
| -250 | +1 | -250 | 250 | 25% | 反转 50% |
| -500 | +1 | -500 | 0 | 0% | 全速反转 |

有效 speed 范围：-500 ~ +500（即 ±PWM_PERIOD/2）。

### 5.3 零点偏移补偿

S8050 三极管上升沿/下降沿不对称（差值约 200~400ns）和 DRV8870 高侧/低侧 RDS(on) 不一致（307mΩ vs 258mΩ，差异 19%）导致实际零速点偏离 50%。

标定流程：

1. 电机空载，逐步调整 duty 值
2. 找到使电机恰好静止的 duty 值
3. zero_duty_offset = 实际静止 duty - PWM_PERIOD/2
4. 典型偏移量：2~5 步（对应 0.2%~0.5% 占空比偏差）

配置位置：`project_config.h` 中的 `PRJ_DRV8870_ZERO_DUTY_OFFSET`。

### 5.4 方向控制

方向通过占空比大小自动实现，无需切换引脚方向。正值 speed → duty > 50% → 正转；负值 speed → duty < 50% → 反转。方向切换是连续过渡，DRV8870 内置 220ns 死区时间处理 H 桥切换，无需软件死区保护。

`dir_sign` 字段用于修正电机安装方向：+1 为默认正方向，-1 为反转方向（电机物理安装方向相反时使用）。

### 5.5 停止与刹车

锁相驱动下，IN2 由硬件反相器驱动，无法独立控制为与 IN1 相同的状态。因此无法实现 IN1=IN2=1 的 Brake 模式或 IN1=IN2=0 的 Coast 模式。

`bsp_drv8870_stop()` 的两种模式均等效为 50% duty 主动阻尼。`mode` 参数保留用于 API 兼容性和未来硬件扩展（如增加独立 IN2 控制 GPIO）。

### 6.1 正确的实现

| 验证项 | 位置 | 结论 |
|-------|------|------|
| 锁相驱动公式 | `speed_to_duty()` L90~93 | duty = 50% + offset + speed，映射无误 |
| 50% duty 初始化 | `bsp_drv8870_init()` L139~143 | 所有通道启动时设为 50% + offset |
| stop 实现 | `bsp_drv8870_stop()` L200~203 | 两种模式均设为 50% duty |
| 无死区保护 | `bsp_drv8870_set_speed()` | 正确省略，锁相驱动方向切换连续 |

### 6.2 发现的问题

#### 问题 1：`get_duty_max()` 返回值错误（P1 中等）

**位置**：`bsp_drv8870.c` 第 216~219 行

```c
uint32_t bsp_drv8870_get_duty_max(void)
{
    return s_drv8870_pwm_period;  // 返回 1000
}
```

**问题**：锁相驱动下有效速度范围是 ±`PWM_PERIOD/2` = ±500，但此函数返回 1000。应用层用此值设置 PID 输出限幅为 ±1000，而 `speed_to_duty()` 内部限幅为 ±500，导致 PID 输出 500~1000 的范围被截断，控制饱和且积分器持续累积。

**修正**：返回 `s_drv8870_pwm_period / 2U`。

#### 问题 2：`set_speed` 头文件注释范围错误（P2 低）

**位置**：`bsp_drv8870.h` 第 112~113 行

```c
 * @param  speed  速度值(0~PWM_PERIOD/2)
```

**问题**：实际接受负值，注释 `0~PWM_PERIOD/2` 会误导调用者认为只能传正值。

**修正**：改为 `-PWM_PERIOD/2 ~ +PWM_PERIOD/2`。

#### 问题 3：SysConfig CC 初始值为 0（P0 严重）

**位置**：`ti_msp_dl_config.c` 第 410/417/424 行

```c
DL_TimerA_setCaptureCompareValue(PWM_MOTOR_INST, 0, DL_TIMER_CC_0_INDEX);
DL_TimerA_setCaptureCompareValue(PWM_MOTOR_INST, 0, DL_TIMER_CC_1_INDEX);
DL_TimerA_setCaptureCompareValue(PWM_MOTOR_INST, 0, DL_TIMER_CC_2_INDEX);
DL_TimerA_setCaptureCompareValue(PWM_MOTOR_INST, 0, DL_TIMER_CC_3_INDEX);
```

**问题**：锁相驱动下 duty=0 = IN1 恒低、IN2 恒高 = 全速反转。从 `SYSCFG_DL_init()` 执行到 `bsp_drv8870_init()` 设置 50% 之间的窗口（约 50~100ms），4 个电机全速反转。

**修正**：所有 CC 初始值从 0 改为 500（= PWM_PERIOD/2）。

#### 问题 4：`percent_to_duty` 语义模糊（P2 低）

**位置**：`bsp_drv8870.c` 第 221~231 行

**问题**：函数将 0~100 线性映射到 0~1000。在锁相驱动下 50%=停止、0%=全速反转、100%=全速正转。如果调用者按 TB6612 习惯认为"percent=0 = 停止"会引发严重错误。

**修正**：改名为 `bsp_drv8870_percent_to_absolute_duty` 并在注释中明确 50%=停止。

#### 问题 5：缺少并发保护（P2 低）

**位置**：`bsp_drv8870_set_speed()` / `bsp_drv8870_stop()`

**问题**：直接调用 `hal_timer_set_pwm_duty()` 写寄存器，无临界区保护。控制任务和菜单任务同时调用时可能产生竞争。单次寄存器写入是原子的，但多通道同步更新时中间可能被打断。

**修正**：在多通道同步更新场景中，调用方使用 `OSAL_CRITICAL_SECTION` 保护（当前应用层已部分实现）。

#### 问题 6：应用层调用错误驱动（P0 严重）

**位置**：`app_main.c`、`task_control.c`、`app_vofa.c`

**问题**：应用层调用 `bsp_motor_*`（TB6612 驱动）而非 `bsp_drv8870_*`。TB6612 驱动的初始化逻辑（PWM=0, IN1=0, IN2=0 → Coast）在 DRV8870 硬件上等效为全速反转；`stop(BRAKE)` 设置 IN1=1/IN2=1 在 DRV8870 上等效为全速正转。

**修正**：替换所有 `bsp_motor_*` 调用为 `bsp_drv8870_*`（见 §7.1）。

#### 问题 7：速度量程不匹配（P1 中等）

**问题**：PID 控制器输出限幅由 `bsp_motor_get_duty_max()` 设置为 ±1000，但 DRV8870 有效速度范围是 ±500。切换驱动后如果 `get_duty_max()` 返回值修正为 500，PID 限幅自动适配；如果不修正则控制饱和。

**修正**：修正 `get_duty_max()` 返回值后自动解决。

### 6.3 问题优先级汇总

| 优先级 | 问题编号 | 描述 | 性质 |
|--------|---------|------|------|
| P0 | #3 | SysConfig CC 初始值为 0 | 上电全速反转 |
| P0 | #6 | 应用层调用 TB6612 驱动 | 驱动与硬件不匹配 |
| P1 | #1 | `get_duty_max()` 返回值错误 | PID 控制饱和 |
| P1 | #7 | 速度量程不匹配 | 控制范围截断 |
| P2 | #2 | speed 范围注释错误 | 文档误导 |
| P2 | #4 | `percent_to_duty` 语义模糊 | 命名歧义 |
| P2 | #5 | 缺少并发保护 | 多任务竞争 |

## 8. 开发核心注意事项

### 8.1 锁相驱动与 sign-magnitude 的关键差异

从 TB6612（sign-magnitude）切换到 DRV8870（锁相驱动）时，以下概念差异必须牢记：

| 概念 | TB6612 (sign-magnitude) | DRV8870 (锁相驱动) |
|------|------------------------|-------------------|
| 停止 | PWM=0 + IN1=IN2=0 | **PWM=50%** |
| 正转 | IN1=1, IN2=0, PWM=duty | PWM > 50% |
| 反转 | IN1=0, IN2=1, PWM=duty | PWM < 50% |
| 刹车 | IN1=IN2=1, PWM=0 | **无法实现**（50% 主动阻尼替代） |
| 滑行 | IN1=IN2=0, PWM=0 | **无法实现** |
| 方向引脚 | 需要 2 个 GPIO | **不需要**（硬件反相器） |
| 死区保护 | 软件实现（先刹车一周期） | **不需要**（H 桥内置 220ns） |
| speed=0 | duty=0 | **duty=50%** |
| speed 范围 | 0~PWM_PERIOD | **-PWM_PERIOD/2 ~ +PWM_PERIOD/2** |

### 8.2 duty 值的含义

锁相驱动下 duty 值的含义与 sign-magnitude 完全不同：

| duty 值 | sign-magnitude 含义 | 锁相驱动含义 |
|---------|--------------------|----|
| 0 | 0% 速度（停止） | **全速反转** |
| 500 | 50% 速度 | **停止** |
| 1000 | 100% 速度 | **全速正转** |

任何将 duty=0 理解为"停止"的代码都是严重错误。初始化必须设置 duty=50% 而非 0。

### 8.3 `get_duty_max()` 的正确用法

`bsp_drv8870_get_duty_max()` 修正后返回 `PWM_PERIOD/2`（500），表示有效速度的绝对值上限。PID 限幅应使用此值：

```c
float speed_max = (float)bsp_drv8870_get_duty_max();  // 500
app_pid_init(&pid, kp, ki, kd, mode, -speed_max, speed_max);
```

PID 输出范围 ±500 直接传给 `bsp_drv8870_set_speed()`，无需额外缩放。

### 8.4 `percent_to_duty` 的陷阱

`bsp_drv8870_percent_to_duty(50)` 返回 500（50% duty = 停止），而非"50% 速度"。此函数映射的是绝对 duty 值，不是速度百分比。如果需要速度百分比，应自行实现：

```c
// 速度百分比 → speed 值
// percent: 0~100, 0=停止, 100=全速
int32_t speed_percent_to_value(uint32_t percent) {
    return (int32_t)(percent * bsp_drv8870_get_duty_max() / 100U);
}
```

### 8.5 开机安全检查清单

每次上电前确认：

1. SysConfig 中 4 个 CC 初始值是否为 500（非 0）
2. `bsp_drv8870_init()` 是否在应用层初始化序列中被调用
3. 电机是否已物理脱开或固定（防止上电瞬间运动造成损坏）
4. VM 电源电压是否在 6.5~45V 范围内
5. 示波器接在 PWM 引脚上，确认上电后输出为 50% 方波

### 8.6 调试技巧

#### 确认锁相驱动是否工作

用示波器同时测量 DRV8870 的 IN1 和 IN2 引脚：

- IN1 和 IN2 应为互补方波（一个高时另一个低）
- 50% duty 时两者对称
- 调整 speed 时 IN1 的高电平时间变化

#### 确认零点是否准确

设置 speed=0，观察电机是否完全静止。如果电机缓慢转动，说明零点偏移未补偿。调整 `PRJ_DRV8870_ZERO_DUTY_OFFSET` 直到电机完全静止。

#### 排查电机异响

锁相驱动在 50% duty 时，电机两端电压以 PWM 频率高速翻转。如果 PWM 频率在可听范围（<20kHz），电机会发出嗡嗡声。当前配置为 20kHz，处于人耳听觉边缘。如果异响明显，可将 PWM 频率提升到 40kHz（修改 SysConfig 中 ARR 从 1000 改为 500）。

#### VOFA+ 波形观测

通过 `app_vofa.c` 输出速度和 duty 值到 VOFA+，观测 PID 控制响应。注意 VOFA+ 输出的 `output` 值是 speed（±500 范围），不是 duty（0~1000 范围）。

### 8.7 常见错误与后果

| 错误操作 | 后果 | 严重程度 |
|---------|------|---------|
| 初始化时 duty=0 | 全速反转 | 严重 |
| `stop()` 设置 duty=0 | 全速反转 | 严重 |
| `get_duty_max()` 返回 1000 | PID 饱和 + 积分累积 | 中等 |
| 方向切换时加死区 | 引入抖动，无实际作用 | 低 |
| 认为 percent=0 = 停止 | 全速反转 | 严重 |
| 修改 SysConfig 后重新生成 | CC 初始值被覆盖回 0 | 严重 |

## 9. PCB 布局要点

### 9.1 热散设计

DRV8870 的 RθJC(bot) = 2.7°C/W，散热依赖 PCB 铜箔：

- PAD 焊盘下方大面积 GND 铜箔 ≥ 100mm²
- ≥ 4 个过孔（孔径 0.3mm）连接顶层和底层 GND
- 底层对应区域铺设 GND 铜箔
- 铜箔厚度建议 2oz（70μm）

### 9.2 功率回路

- VM → DRV8870 → OUT1/OUT2 → 电机 → ISEN → R12 → GND 回路面积最小化
- VM 去耦电容紧邻 Pin 5（VM）和 Pin 1（GND）
- 检流电阻 R12 紧邻 Pin 7（ISEN）和 GND
- OUT1/OUT2 走线宽度 ≥ 3.6mm（按 1A/mm 规则，3.6A 需 ≥ 3.6mm）

### 9.3 信号完整性

- PWM 信号走线远离 OUT1/OUT2 和 VM 走线
- ADC 采样线使用差分走线或地保护环
- 运放电路靠近检流电阻，走线长度 < 10mm
- 编码器信号线远离电机驱动区域

### 9.4 VM 电源去耦

数据手册要求 VM 连接 0.1μF 旁路电容和足够的大容量电容。当前仅有 100nF，需增加：

| 电容类型 | 推荐值 | 位置 |
|---------|--------|------|
| 陶瓷电容 | 100nF | 紧邻 VM 引脚（已有） |
| 陶瓷电容 | 10μF (X7R) | VM 入口处 |
| 电解电容 | 100μF | VM 总线入口 |

### 9.5 检流电阻封装

| 工作状态 | R12 功耗 | 所需封装 |
|---------|---------|---------|
| 限流点 2.2A | 0.73W | ≥ 1W（2010/2512） |
| 峰值 3.6A | 1.94W | ≥ 2W（2512） |

推荐：2512 封装，额定功率 ≥ 1W，精度 ±1%。
