## 1. 系统概述

本系统为 4 通道直流有刷电机驱动方案，采用 TI DRV8870DDAR 作为 H 桥驱动芯片，配合 S8050 NPN 三极管反相器实现单 PWM 锁相驱动（Locked Anti-Phase）控制。每个通道配备独立的电流采样电路（LMV321 运放 + 150mΩ 检流电阻），通过 ADC 反馈至 MCU 实现软件电流监控。

### 1.1 系统架构

```
┌──────────┐    Mx_PWM1     ┌──────────┐    Mx_PWM2     ┌──────────┐
│          │────────────────┼─→ IN1     │                │          │
│   MCU    │                │ DRV8870  │                │  直流    │
│          │    ┌───────┐   │          │   OUT1 ────────│  电机    │
│  PA8/9/  │────│S8050  │───┼─→ IN2     │   OUT2 ────────│          │
│  PB17/2  │    │反相器 │   │          │                │          │
│          │    └───────┘   └────┬─────┘                └──────────┘
│          │                     │
│          │    ISEN ──→ R12(150mΩ) ──→ GND
│          │                     │
│          │    ┌───────┐        │
│  ADC     │◄───│LMV321 │◄───────┘
│  PA15-22 │    │×10放大│
│          │    └───────┘
└──────────┘
```

## 2. 硬件部分

### 2.1 引脚定义与连接

#### 2.1.1 DRV8870 引脚定义

DRV8870DDAR 采用 HSOP-8 封装（带散热焊盘），引脚定义依据 TI 数据手册 SLVSCY8B：

| 引脚号 | 引脚名 | 类型 | 功能描述 | 本项目连接 |
|-------|--------|------|---------|-----------|
| 1 | GND | PWR | 逻辑地 | GND |
| 2 | IN2 | I | 逻辑输入 2，内部下拉 100kΩ | Mx_PWM2（经 S8050 反相） |
| 3 | IN1 | I | 逻辑输入 1，内部下拉 100kΩ | Mx_PWM1（MCU 直连） |
| 4 | VREF | I | 电流调节参考电压（0.3~5V） | 3V3 |
| 5 | VM | PWR | 电机电源输入（6.5~45V） | VIN_OUT |
| 6 | OUT1 | O | H 桥输出 1 | Mx_OUT1 → 电机端 A |
| 7 | ISEN | PWR | 电流检测引脚 | R12(150mΩ) → GND |
| 8 | OUT2 | O | H 桥输出 2 | Mx_OUT2 → 电机端 B |
| PAD | — | — | 散热焊盘 | GND（大面积铜箔） |

#### 2.1.2 S8050 反相器电路

每个通道使用 1 个 S8050 NPN 三极管构建共射极反相器，将 MCU 单路 PWM 信号转换为互补信号对：

| 元件 | 参数 | 功能 |
|------|------|------|
| Q1~Q4 | S8050 (0.5A) | NPN 三极管，共射极反相 |
| 基极限流电阻 | 1kΩ | 限制基极电流 |
| 基极下拉电阻 | 10kΩ | 确保 MCU 高阻态时三极管截止 |
| 集电极上拉电阻 | 1kΩ | 三极管截止时拉高输出 |

信号链路：

```
MCU GPIO ──→ Mx_PWM1 ──┬──→ DRV8870 IN1 (直连)
                        │
                        └──→ [1kΩ] ──→ S8050 基极
                                          │
                                     S8050 反相
                                          │
                         Mx_PWM2 ←──[1kΩ上拉]──← S8050 集电极 ──→ DRV8870 IN2
```

#### 2.1.3 电流采样电路

| 元件 | 参数 | 功能 |
|------|------|------|
| R12/R18/R16/R28 | 150mΩ | 检流电阻（串联在 ISEN 与 GND 之间） |
| R4/R6/R14/R19 | 10kΩ | 运放反相输入电阻 |
| R5/R8/R15/R20 | 90kΩ | 运放反馈电阻 |
| U7/U10/U9/U12 | LMV321IDBVR | 单通道轨到轨运放，SOT-23-5 |
| C5/C6/C7/C9 | 100nF | 运放输入滤波电容 |

运放增益计算：

```
Av = 1 + R_feedback / R_input = 1 + 90kΩ / 10kΩ = 10 倍
```

#### 2.1.4 电源网络

| 网络名 | 电压 | 用途 | 去耦电容 |
|--------|------|------|---------|
| 3V3 | 3.3V | 逻辑供电、VREF、运放供电、S8050 上拉 | 100nF × 4（每通道） |
| VIN_OUT | 6.5~45V | DRV8870 VM 电机电源 | 100nF × 4 + 需增加大容量电容 |
| GND | 0V | 系统地 | — |

### 2.2 原理图说明

#### 2.2.1 DRV8870 H 桥控制真值表

依据数据手册 Table 1：

| IN1 | IN2 | OUT1 | OUT2 | 模式 | 说明 |
|-----|-----|------|------|------|------|
| 0 | 0 | High-Z | High-Z | Coast（滑行） | 1ms 后进入休眠模式 |
| 0 | 1 | L | H | Reverse（反转） | 电流方向 OUT2 → OUT1 |
| 1 | 0 | H | L | Forward（正转） | 电流方向 OUT1 → OUT2 |
| 1 | 1 | L | L | Brake（刹车） | 低侧慢衰减 |

#### 2.2.2 锁相驱动工作原理

本系统采用锁相驱动（Locked Anti-Phase）拓扑。MCU 输出单路 PWM 信号，直连 DRV8870 IN1，同时经 S8050 反相后接入 IN2。因此 IN1 和 IN2 始终互补：

| PWM 占空比 | IN1 状态 | IN2 状态 | 平均输出电压 | 电机行为 |
|-----------|---------|---------|------------|---------|
| 50% | 1/0 交替 | 0/1 交替 | 0V | 停止（主动阻尼） |
| >50% | 1 的时间长 | 0 的时间长 | >0V | 正转 |
| <50% | 0 的时间长 | 1 的时间长 | <0V | 反转 |
| 100% | 恒 1 | 恒 0 | +VM | 全速正转 |
| 0% | 恒 0 | 恒 1 | -VM | 全速反转 |

平均电压公式：

```
V_avg = (2 × D - 1) × VM

其中 D 为占空比（0~1），VM 为电机电源电压
```

50% 占空比时，电机两端电压在 +VM 和 -VM 之间高速翻转。若外力迫使电机转动，反电动势与当前驱动方向叠加，产生抵抗扭矩，表现为主动阻尼（Active Damping）效果，电机呈现很强的保持力矩。

#### 2.2.3 Buck 变换器类比

锁相驱动的电机回路可类比为不带输出电容的 Buck 变换器：

| Buck 变换器组件 | 电机驱动对应物 |
|----------------|--------------|
| 开关管 MOSFET | H 桥（DRV8870） |
| 续流二极管 | H 桥对侧 MOS 管体二极管 |
| 滤波电感 L | 电机线圈电感 |
| 滤波电容 + 负载 | 机械惯性 + 反电动势 |

纹波电流公式（50% 占空比，电机静止时）：

```
ΔI_pp = VM / (2 × L × f_pwm)

其中 VM 为电机电源电压，L 为电机电感量，f_pwm 为 PWM 频率
```

此公式在以下假设条件下成立：电机静止（反电动势为零）、电感时间常数远大于 PWM 半周期、忽略电机内阻对纹波的衰减。

### 2.3 PCB 布局要点

#### 2.3.1 热散设计

DRV8870 的 RθJC(bot) = 2.7°C/W（结到封装底部），散热严重依赖 PCB 铜箔：

- PAD 焊盘下方布置大面积 GND 铜箔，建议 ≥ 100mm²
- 使用多个过孔（≥ 4 个，孔径 0.3mm）连接顶层和底层 GND 平面
- 底层对应区域同样铺设 GND 铜箔，通过过孔阵列连接
- 铜箔厚度建议 2oz（70μm）以降低热阻

#### 2.3.2 功率回路布局

电机驱动的大电流回路需要最小化寄生电感：

- VM → DRV8870 → OUT1/OUT2 → 电机 → OUT2/OUT1 → ISEN → R12 → GND 回路面积最小化
- VM 去耦电容（100nF）尽量靠近 Pin 5（VM）和 Pin 1（GND）
- 检流电阻 R12 尽量靠近 Pin 7（ISEN）和 GND
- OUT1/OUT2 走线宽度 ≥ 1mm（按 1A/mm 规则，3.6A 需 ≥ 3.6mm）

#### 2.3.3 信号完整性

- PWM 信号走线（Mx_PWM1/Mx_PWM2）远离功率输出走线（OUT1/OUT2）和 VM 走线
- ADC 采样线（ADC_SENSE1~4）使用差分走线或地保护环
- 运放电路靠近检流电阻放置，走线长度 < 10mm
- 编码器信号线（Mx_A/Mx_B）远离电机驱动区域，必要时加滤波

#### 2.3.4 VM 电源去耦

数据手册要求 VM 引脚连接 0.1μF 旁路电容和足够的大容量电容。当前原理图仅有 100nF，缺少大容量储能电容。PCB 布局时应预留位置：

| 电容类型 | 推荐值 | 用途 | 位置 |
|---------|--------|------|------|
| 陶瓷电容 | 100nF | 高频去耦 | 紧邻 VM 引脚 |
| 陶瓷电容 | 10μF (X7R) | 中频去耦 | VM 入口处 |
| 电解电容 | 100μF | 储能/低频 | VM 总线入口 |

#### 2.4.4 电流调节功能

DRV8870 内置电流调节功能，通过 VREF 引脚和 ISEN 引脚外接检流电阻实现：

```
I_TRIP = VREF / (AV × R_SENSE)

其中:
  VREF = 3.3V（本设计接 3V3）
  AV = 10 V/V（数据手册典型值，范围 9.4~10.4）
  R_SENSE = 150mΩ = 0.15Ω（R12 等检流电阻）

I_TRIP = 3.3 / (10 × 0.15) = 2.2A（典型值）
I_TRIP_min = 3.3 / (10.4 × 0.15) = 2.12A
I_TRIP_max = 3.3 / (9.4 × 0.15) = 2.34A
```

电流调节参数（数据手册值）：

| 参数 | 符号 | 典型值 | 单位 |
|------|------|--------|------|
| ISEN 增益 | AV | 10 | V/V |
| PWM 关断时间 | tOFF | 25 | μs |
| PWM 消隐时间 | tBLANK | 2 | μs |

#### 2.4.5 电流采样电路参数

## 3. 软件部分

### 3.1 初始化配置

#### 3.1.1 PWM 定时器配置

以 M1 通道（PA8）为例，使用定时器输出 PWM 信号：

```c
// PWM 频率: 40kHz
// 定时器时钟: 80MHz
// 预分频: PSC = 0 (不分频)
// 自动重装载: ARR = 80MHz / 40kHz - 1 = 1999
// 初始占空比: CCR = ARR / 2 = 1000 (50%，电机停止)

#define PWM_FREQUENCY   40000   // 40kHz
#define PWM_ARR         1999    // 80MHz / 40kHz - 1
#define PWM_DUTY_ZERO   1000    // 50% 占空比 = 零速

void Motor_PWM_Init(void) {
    // 1. 配置 GPIO 为复用功能
    // 2. 配置定时器时基: PSC=0, ARR=1999
    // 3. 配置输出比较: PWM Mode 1, OCxPE=Enable
    // 4. 设置 CCR = PWM_DUTY_ZERO (50% 占空比)
    // 5. 使能定时器主输出 (MOE)
    // 6. 使能通道输出

    __HAL_TIM_SET_COMPARE(&htim1, TIM_CHANNEL_1, PWM_DUTY_ZERO);
    HAL_TIM_PWM_Start(&htim1, TIM_CHANNEL_1);
}
```

PWM 分辨率计算：

```
分辨率步数 = 定时器时钟 / PWM 频率 = 80MHz / 40kHz = 2000 步
有效分辨率 = log2(2000) ≈ 10.97 位
```

各通道引脚与定时器映射需根据具体 MCU 型号确认。

#### 3.1.2 ADC 电流采样配置

```c
#define ADC_VREF        3.3f    // ADC 参考电压
#define ADC_RESOLUTION  4096    // 12 位 ADC
#define AMP_GAIN        10.0f   // 运放增益
#define SENSE_RESISTOR  0.15f   // 检流电阻 150mΩ

// 电流计算公式:
// I = ADC_raw × VREF / ADC_RESOLUTION / AMP_GAIN / SENSE_RESISTOR
// I = ADC_raw × 3.3 / 4096 / 10 / 0.15
// I = ADC_raw × 0.0005371 (A)
// I = ADC_raw × 0.5371 (mA)

#define ADC_TO_CURRENT_MA(adc_raw) ((float)(adc_raw) * 3.3f / 4096.0f / 10.0f / 0.15f * 1000.0f)

// ADC 配置要点:
// - 采样时间 ≥ 15 个 ADC 时钟周期（确保低阻抗采样）
// - 4 个通道使用 DMA 循环模式自动采集
// - 采样频率建议 ≥ 1kHz（10ms 周期内至少 10 次采样）
// - 启用 ADC 过采样（如支持）提高信噪比
```

#### 3.1.3 开机安全初始化（关键）

由于锁相驱动拓扑的固有特性，MCU 复位期间 GPIO 处于高阻态时，DRV8870 的 IN1 被内部下拉为 0，IN2 被 S8050 上拉电阻拉高为 1，导致 IN1=0/IN2=1 → 电机全速反转。

此问题无法通过软件完全消除（问题发生在代码执行之前），但可以通过以下措施缩短危险窗口：

```c
// 措施 1: 在启动汇编阶段尽早配置 GPIO
// 文件: startup_stm32xxxx.s (或对应 MCU 启动文件)
// 在 Reset_Handler 中，调用 SystemInit 之前插入:

// 将 PWM 引脚配置为推挽输出，初始输出 HIGH
// 此时 IN1=1, IN2=0 → 正转（非反转），后续快速切到 50% 停止
// 危险窗口从 50~100ms 缩短到 2~5ms（仅 POR + 时钟启动时间）

// 措施 2: 定时器预配置 50% 占空比
void Motor_Early_Init(void) {
    // 在使能定时器输出之前:
    // 1. 配置 GPIO 为复用功能
    // 2. 配置定时器时基和输出比较
    // 3. 设置 CCR = ARR/2 (50% 占空比)
    // 4. 使能定时器
    // 确保第一个 PWM 脉冲就是 50% 占空比

    __HAL_TIM_SET_COMPARE(&htim1, TIM_CHANNEL_1, PWM_DUTY_ZERO);
    // 然后才使能输出
}
```

### 3.2 PWM 控制逻辑

#### 3.2.1 占空比到速度映射

锁相驱动下，占空比 D 与速度的映射关系：

```
D = 0%   → V_avg = -VM → 全速反转
D = 50%  → V_avg = 0V  → 停止（主动阻尼）
D = 100% → V_avg = +VM → 全速正转
```

软件实现中使用 0~ARR 范围的整数值表示占空比：

```c
// 占空比映射宏定义
// target: -1.0(全速反转) ~ 0.0(停止) ~ +1.0(全速正转)

#define ZERO_DUTY_OFFSET    0   // 零点偏移补偿（需实测标定，典型值 0~5 步）

int16_t Motor_SpeedToDuty(float target) {
    // target 限幅
    if (target > 1.0f)  target = 1.0f;
    if (target < -1.0f) target = -1.0f;

    // 映射到 CCR 值
    int16_t duty = (int16_t)(PWM_DUTY_ZERO + ZERO_DUTY_OFFSET
                  + target * PWM_DUTY_ZERO);

    // 范围保护
    if (duty < 0)             duty = 0;
    if (duty > PWM_ARR)       duty = PWM_ARR;

    return duty;
}
```

#### 3.2.2 死区补偿

由于 S8050 三极管上升沿/下降延不对称（差值约 200~400ns），以及 DRV8870 高侧/低侧 RDS(on) 不一致（307mΩ vs 258mΩ，差异 19%），实际零速点可能偏离 50% 占空比 1%~3%。

```c
// 标定流程:
// 1. 电机空载，逐步调整占空比
// 2. 找到使电机恰好静止的 CCR 值
// 3. 记录偏移量: ZERO_DUTY_OFFSET = 实际静止点 - PWM_DUTY_ZERO
// 4. 典型偏移量: 2~5 步（对应 0.1%~0.25% 占空比偏差）

#define ZERO_DUTY_OFFSET    3   // 标定后的零点偏移（示例值）

// 补偿后的占空比计算已在 Motor_SpeedToDuty() 中实现
```

#### 3.2.3 软启动斜坡控制

避免占空比突变导致电流冲击，限制占空比变化速率：

```c
#define RAMP_STEP_MAX       (PWM_ARR / 50)  // 每步最大变化量 = 40 步 ≈ 2% 占空比
#define RAMP_PERIOD_MS      10              // 斜坡更新周期 10ms
// 最大加速度: 2%/10ms = 200%/s

static int16_t current_duty[4] = {
    PWM_DUTY_ZERO, PWM_DUTY_ZERO,
    PWM_DUTY_ZERO, PWM_DUTY_ZERO
};

void Motor_Ramp_Update(uint8_t channel, int16_t target_duty) {
    int16_t diff = target_duty - current_duty[channel];

    if (diff > RAMP_STEP_MAX) {
        current_duty[channel] += RAMP_STEP_MAX;
    } else if (diff < -RAMP_STEP_MAX) {
        current_duty[channel] -= RAMP_STEP_MAX;
    } else {
        current_duty[channel] = target_duty;
    }

    // 写入定时器 CCR
    Motor_SetDuty(channel, current_duty[channel]);
}
```

#### 3.2.4 PWM 频率选择

| PWM 频率 | 分辨率 | 纹波电流 | 开关损耗 | 推荐场景 |
|---------|--------|---------|---------|---------|
| 20kHz | 4000 步 (12 位) | 较大 | 低 | 低电感电机 |
| 40kHz | 2000 步 (11 位) | 中等 | 中等 | 通用推荐 |
| 100kHz | 800 步 (9.6 位) | 小 | 高 | 高电感电机 |

本项目推荐 40kHz：在分辨率（11 位）和开关损耗之间取得平衡，且超出人耳听觉范围。

### 3.3 正反转与刹车实现

#### 3.3.1 正反转控制

在锁相驱动模式下，正反转通过占空比大小自动实现，无需切换引脚方向：

```c
typedef enum {
    MOTOR_DIR_FORWARD  =  1,
    MOTOR_DIR_STOP     =  0,
    MOTOR_DIR_REVERSE  = -1,
} Motor_Direction_t;

void Motor_SetSpeed(uint8_t channel, float speed) {
    // speed: -1.0 ~ 0 ~ +1.0
    // 正值 = 正转, 负值 = 反转, 0 = 停止
    int16_t target_duty = Motor_SpeedToDuty(speed);
    Motor_Ramp_Update(channel, target_duty);
}
```

#### 3.3.2 主动阻尼（50% 停止）

设置占空比为 50% 时，电机两端电压在 +VM 和 -VM 之间高速翻转，产生主动阻尼效果：

```c
void Motor_Stop(uint8_t channel) {
    // 软停止: 斜坡回到 50% 占空比
    Motor_Ramp_Update(channel, PWM_DUTY_ZERO + ZERO_DUTY_OFFSET);
}

void Motor_EmergencyStop(uint8_t channel) {
    // 紧急停止: 立即设置 50% 占空比（无斜坡）
    current_duty[channel] = PWM_DUTY_ZERO + ZERO_DUTY_OFFSET;
    Motor_SetDuty(channel, current_duty[channel]);
}
```

#### 3.3.3 硬件刹车（Brake）

若需要更强的刹车效果，可暂时退出锁相驱动模式，将 IN1 和 IN2 同时置高实现低侧刹车。但本硬件拓扑中 IN2 由反相器驱动，无法独立控制为与 IN1 相同的状态。因此硬件刹车需要额外 GPIO 控制 VREF 或 VM 电源。

在当前硬件下，刹车只能通过锁相驱动的主动阻尼实现。若需更强的刹车力，可将占空比快速从当前值斜坡过渡到 50%，利用反向电流进行电磁制动。

### 3.4 故障检测与保护机制

#### 3.4.1 软件过流保护

通过 ADC 采样电流检测放大器输出，实现软件级电流监控：

```c
#define CURRENT_LIMIT_A        2.0f    // 软件限流阈值 2.0A
#define CURRENT_STALL_A        1.5f    // 堵转检测阈值 1.5A
#define CURRENT_FAULT_A        2.5f    // 故障阈值 2.5A
#define OVERCURRENT_COUNT_MAX  10      // 连续过流计数上限

static uint8_t overcurrent_counter[4] = {0};
static uint8_t motor_fault[4] = {0};

void Motor_Current_Protection(uint8_t channel) {
    uint16_t adc_raw = ADC_ReadChannel(channel);
    float current = ADC_TO_CURRENT_MA(adc_raw) / 1000.0f;  // 转换为安培

    if (current > CURRENT_FAULT_A) {
        // 严重过流: 立即停止电机
        Motor_EmergencyStop(channel);
        motor_fault[channel] = FAULT_OVERCURRENT;
        overcurrent_counter[channel] = OVERCURRENT_COUNT_MAX;
    } else if (current > CURRENT_LIMIT_A) {
        // 轻度过流: 计数累计
        overcurrent_counter[channel]++;
        if (overcurrent_counter[channel] >= OVERCURRENT_COUNT_MAX) {
            Motor_EmergencyStop(channel);
            motor_fault[channel] = FAULT_OVERCURRENT_SUSTAINED;
        } else {
            // 限制占空比向 50% 方向回调
            int16_t current_duty_val = current_duty[channel];
            int16_t zero_duty = PWM_DUTY_ZERO + ZERO_DUTY_OFFSET;
            current_duty_val = (current_duty_val + zero_duty) / 2;
            Motor_SetDuty(channel, current_duty_val);
        }
    } else {
        // 电流正常: 清除计数
        if (overcurrent_counter[channel] > 0) {
            overcurrent_counter[channel]--;
        }
    }
}
```

#### 3.4.2 DRV8870 硬件保护链

DRV8870 内置三级硬件保护，无需软件干预：

| 保护类型 | 触发条件 | 触发动作 | 恢复方式 |
|---------|---------|---------|---------|
| 欠压锁定 (UVLO) | VM < 6.1V | 输出 High-Z | VM > 6.3V 自动恢复 |
| 过流保护 (OCP) | 输出电流 > 3.7~6.4A | 输出 High-Z | 3ms 后自动重试 |
| 热关断 (TSD) | 结温 > 175°C | 输出 High-Z | 温度下降 40°C 后自动恢复 |
| 电流调节 | ISEN 压降 > VREF/AV | 电流切断 25μs | 25μs 后自动恢复 |

电流调节与过流保护的关系：

```
正常工作: I < 2.2A (VREF 限流点) → 无干预
电流调节: 2.2A < I < 3.7A → DRV8870 自动限流 (25μs off-time)
过流保护: I > 3.7A → DRV8870 输出 High-Z (3ms 重试)
软件保护: I > 2.0A → MCU 限制占空比 (优先级最高，最先触发)
```

软件保护阈值（2.0A）低于硬件限流点（2.2A），确保软件保护优先于硬件干预。

#### 3.4.3 堵转检测

通过电流持续高于阈值且编码器无转速变化来判定堵转：

```c
#define STALL_CURRENT_THRESHOLD  1.5f   // 堵转电流阈值
#define STALL_SPEED_THRESHOLD    10     // 堵转速度阈值 (编码器 count/10ms)
#define STALL_TIME_MS            500    // 堵转持续时间阈值

typedef struct {
    uint8_t  channel;
    float    current;
    int16_t  encoder_speed;
    uint32_t stall_timer;
    uint8_t  stall_detected;
} Motor_StallDetect_t;

void Motor_Stall_Check(Motor_StallDetect_t *motor) {
    if (motor->current > STALL_CURRENT_THRESHOLD
        && abs(motor->encoder_speed) < STALL_SPEED_THRESHOLD) {

        motor->stall_timer += CONTROL_PERIOD_MS;  // 控制周期 (如 10ms)

        if (motor->stall_timer >= STALL_TIME_MS) {
            motor->stall_detected = 1;
            Motor_EmergencyStop(motor->channel);
            // 上报堵转故障
        }
    } else {
        motor->stall_timer = 0;
        motor->stall_detected = 0;
    }
}
```

#### 3.4.4 故障处理状态机

```c
typedef enum {
    STATE_NORMAL,           // 正常运行
    STATE_CURRENT_LIMIT,    // 电流限制中
    STATE_STALL,            // 堵转保护
    STATE_FAULT,            // 严重故障
    STATE_RECOVERY,         // 故障恢复中
} Motor_State_t;

void Motor_StateMachine(uint8_t channel) {
    static Motor_State_t state[4] = {STATE_NORMAL, STATE_NORMAL,
                                     STATE_NORMAL, STATE_NORMAL};
    static uint32_t recovery_timer[4] = {0};

    switch (state[channel]) {
        case STATE_NORMAL:
            if (motor_fault[channel] != 0) {
                state[channel] = STATE_FAULT;
            } else if (overcurrent_counter[channel] > 0) {
                state[channel] = STATE_CURRENT_LIMIT;
            }
            break;

        case STATE_CURRENT_LIMIT:
            Motor_Current_Protection(channel);
            if (overcurrent_counter[channel] == 0) {
                state[channel] = STATE_NORMAL;
            } else if (motor_fault[channel] != 0) {
                state[channel] = STATE_FAULT;
            }
            break;

        case STATE_FAULT:
            Motor_EmergencyStop(channel);
            recovery_timer[channel] = FAULT_RECOVERY_DELAY_MS;
            state[channel] = STATE_RECOVERY;
            break;

        case STATE_RECOVERY:
            if (recovery_timer[channel] > 0) {
                recovery_timer[channel] -= CONTROL_PERIOD_MS;
            } else {
                motor_fault[channel] = 0;
                overcurrent_counter[channel] = 0;
                current_duty[channel] = PWM_DUTY_ZERO + ZERO_DUTY_OFFSET;
                state[channel] = STATE_NORMAL;
            }
            break;
    }
}
```

## 4. 已知问题与缓解措施

### 4.1 开机暴走风险（P0 严重）

**问题描述**: MCU 复位期间 GPIO 处于高阻态，DRV8870 IN1 被内部下拉（100kΩ）为低，IN2 被 S8050 上拉电阻（1kΩ）拉高，导致 IN1=0/IN2=1 → 电机全速反转。4 个通道同时受影响。

**根因**: 锁相驱动拓扑中反相器的固有特性，不是器件缺陷。

**软件缓解措施**:

| 措施 | 实现方式 | 效果 | 残余风险 |
|------|---------|------|---------|
| 早期 GPIO 配置 | 启动汇编中直接操作寄存器，配置 PWM 引脚为推挽输出 HIGH | 危险窗口缩短至 2~5ms | POR+时钟启动期间仍有全速反转 |
| 预配置 50% PWM | 定时器使能前设置 CCR=ARR/2 | 首个 PWM 脉冲即为 50% | 首个脉冲之前仍有风险 |
| 软启动斜坡 | 从 50% 开始逐步调节 | 避免启动电流冲击 | 不影响开机问题 |

**硬件修正方案**（若可修改硬件）:

| 方案 | 改动量 | 效果 |
|------|-------|------|
| 上拉电阻供电端串 P-MOSFET | 增加 4 个 P-MOSFET + 4 个 GPIO | MCU 启动后才给上拉供电，彻底消除 |
| 改用 74LVC1G14 + EN 信号 | 替换 4 个三极管 + 增加 EN 控制 | 施密特反相 + 使能控制 |
| 放弃锁相驱动，改用双 PWM | 删除反相器，MCU 输出两路独立 PWM | 无开机风险，但占用双倍 GPIO |
| VM 电源加 MOSFET 开关 | 增加 1 个 P-MOSFET 控制 VM | MCU 启动后再接通 VM |

### 4.2 VM 缺少大容量储能电容（P0 严重）

**问题描述**: 原理图中 VM 引脚仅有 100nF 陶瓷电容，缺少大容量储能电容。电机换向时瞬态电流可达数安培，会导致 VM 电压塌陷和纹波剧增。

**数据手册要求**: "Connect a 0.1μF bypass capacitor to ground, as well as sufficient bulk capacitance, rated for the VM voltage."

**修正方案**: 在 VM 附近增加 10μF (X7R 陶瓷) + 100μF (电解) 电容，耐压 ≥ VM × 1.5。

### 4.3 S8050 开关不对称（P1 中等）

**问题描述**: S8050 上升沿（截止→导通）通过 1kΩ 上拉充电，延迟 300~500ns；下降沿（导通→截止）通过三极管放电，延迟 100~200ns。不对称量约 200~400ns，在 40kHz 下占周期 1%~2%。

**影响**: 实际零速点偏离 50% 占空比，需软件标定补偿。

**软件缓解**: 通过 `ZERO_DUTY_OFFSET` 参数标定补偿。

**硬件修正**: 替换为 74LVC1G14 施密特反相器（SOT-23-5），传播延迟 3~5ns，上升/下降沿对称。

### 4.4 电流采样放大器量程余量不足（P1 中等）

**问题描述**: 运放增益 10 倍，满量程对应 2.2A，与硬件限流点 2.12~2.34A 几乎重合。LMV321 轨到轨余量约 0.1V，实际最大可测电流约 2.13A，存在测量盲区。

**修正方案**: 将反馈电阻 R5 从 90kΩ 改为 82kΩ（增益 9.2 倍），满量程对应约 2.4A，留出 0.2A 余量。

### 4.5 检流电阻功率等级（P1 中等）

**问题描述**: 限流点 2.2A 时 R12 功耗 0.73W，峰值 3.6A 时 1.94W。若使用 0603 封装（额定 0.1W）会烧毁。

**修正方案**: 使用 2512 封装，额定功率 ≥ 1W，精度 ±1%。

### 4.6 网络标签一致性（P2 低）

**问题描述**: 三极管反相器输入标注为 `Mx_PWM`，而 MCU 和 DRV8870 端标注为 `Mx_PWM1`，可能为同一网络的简写，但需在原理图中统一或确认连接关系。

### 4.7 传播延迟未补偿（P2 低）

**问题描述**: DRV8870 传播延迟 tPD = 0.7~1μs，在 40kHz 下占周期 4%，影响锁相驱动的零点精度。

**软件缓解**: 纳入 `ZERO_DUTY_OFFSET` 标定范围，实测补偿。

### 4.8 问题优先级汇总

| 优先级 | 问题 | 性质 | 解决方式 |
|--------|------|------|---------|
| P0 | 开机暴走风险 | 硬件拓扑缺陷 | 软件缓解 + 硬件修正 |
| P0 | VM 缺少大容量电容 | 硬件设计遗漏 | 硬件修正 |
| P1 | S8050 开关不对称 | 器件特性 | 软件补偿 / 硬件替换 |
| P1 | 运放量程余量不足 | 参数设计 | 硬件修正（改 R5） |
| P1 | 检流电阻功率等级 | 封装选型 | 硬件修正（改封装） |
| P2 | 网络标签一致性 | 文档规范 | 原理图修正 |
| P2 | 传播延迟未补偿 | 器件特性 | 软件标定补偿 |
