# 公共接口参考 v0.1.0

本文只列项目自研公共接口；TI DriverLib、FreeRTOS原生API及LSM6DSR寄存器级内部函数不重复展开。外设配置路径和更完整示例另见《外设分类开发参考手册.md》。

## 1. 应用入口与共享状态

文件：`Application/app_main.h`

- `int32_t app_main_init(void)`：初始化BSP、算法和三个任务；0成功，负值表示模块或任务创建失败。
- `void app_motor_stop(ctx, motor_idx)`：禁用指定电机、复位PID并写中性停止；`motor_idx=0..3`。
- `void app_motor_stop_all(ctx)`：四路统一停止。
- `app_shared_ctx_t`：包含四路PID、前馈、电机使能、RPM/输出状态、IMU状态和位置控制器。跨任务访问必须使用OSAL临界区并尽量只复制快照。

## 2. 统一电机接口 `bsp_motor`

- 公共业务文件：`BSP/Peripherals/bsp_motor.h`；
- 编译期后端：`BSP/Peripherals/bsp_motor.c`；
- 芯片后端：`bsp_drv8870.[ch]`（默认）、`bsp_tb6612.[ch]`（备用）；
- 配置：`Config/project_config.h` 的 `PRJ_MOTOR_DRIVER`、`PRJ_MOTOR_*`、`PRJ_DRV8870_*`、`PRJ_TB6612_*`。

应用层、控制算法和协议处理只允许使用本节公共接口，不直接依赖具体芯片。

### 2.1 类型与公共约定

- `bsp_motor_id_t`：`BSP_MOTOR_A..D`，对应 M1右后、M2右前、M3左前、M4左后；
- `bsp_motor_stop_mode_t`：`BSP_MOTOR_MODE_COAST/BRAKE`；后端不支持时安全降级；
- `bsp_motor_driver_t`：当前编译后端；
- `bsp_motor_capability_t`：功率闸门、真实Coast、真实Brake、锁相驱动、方向GPIO能力位；
- 统一命令范围：`-PRJ_MOTOR_COMMAND_MAX..+PRJ_MOTOR_COMMAND_MAX`，当前为 `-500..+500`；
- 正命令始终表示车体前进方向，安装差异由 `PRJ_MOTOR_x_INSTALL_DIR_SIGN` 修正。

### 2.2 核心API

| API | 参数/合法值 | 说明 |
|---|---|---|
| `bsp_motor_init()` | 无；`SYSCFG_DL_init()`之后调用 | 初始化所选后端并保持安全停止，不自动开启功率 |
| `bsp_motor_power_enable()` | 已初始化 | 以安全输出为前置条件开启PB19、STBY或软件运行闸门 |
| `bsp_motor_power_disable()` | 可在未初始化/故障清理路径调用 | 先停止全部电机，再关闭后端功率/待机控制 |
| `bsp_motor_power_is_enabled()` | 无 | 返回软件命令状态；不等于物理电压反馈 |
| `bsp_motor_set_speed(motor,command)` | `motor=A..D`；命令内部限幅到`[-500,+500]` | 设置车体方向一致的有符号电机命令；功率未开启返回`BSP_ERR_NOT_READY` |
| `bsp_motor_stop(motor,mode)` | 合法ID与COAST/BRAKE | TB6612可实现真实状态；当前DRV8870均降级为50%中性 |
| `bsp_motor_stop_all()` | 无 | 使用BRAKE语义停止全部；不自动关闭功率 |
| `bsp_motor_get_command_max()` | 初始化后调用 | 当前返回500；未初始化返回0 |
| `bsp_motor_percent_to_command(percent)` | 0..100；超界钳位到100 | 百分比换算为统一命令幅值，不包含方向 |
| `bsp_motor_get_driver/name()` | 无 | 返回编译期后端枚举和稳定英文名称 |
| `bsp_motor_get_capabilities()` | 无 | 返回所选后端能力位，业务不得仅按停止枚举名称推断能力 |

典型调用：

```c
#include "bsp_motor.h"
#include "project_config.h"
#include "osal.h"

void motor_demo(void)
{
    if (bsp_motor_power_enable() != BSP_OK) {
        return;
    }

    osal_task_delay_ms(PRJ_MOTOR_POWER_STARTUP_MS);

    if (bsp_motor_set_speed(BSP_MOTOR_A, +100) != BSP_OK) {
        bsp_motor_power_disable();
        return;
    }

    osal_task_delay_ms(300U);
    bsp_motor_stop_all();
    osal_task_delay_ms(PRJ_MOTOR_POWER_SETTLE_MS);
    bsp_motor_power_disable();
}
```

### 2.3 默认DRV8870后端

- 当前硬件为单PWM锁相驱动，40%～55%（含边界）是实测停止死区，50%为中性；
- 非零命令会严格映射到 `<40%` 或 `>55%`；
- PB19为高有效总电源门控；
- 不能产生真正的Coast/Brake；
- 应用层不得调用普通 `bsp_drv8870_set_speed/stop/power_*`，这些只供门面实现使用。

DRV8870专属 `bsp_drv8870_hw_scope_*()` 仅允许 `Application/app_debug.c` 的FactoryTest示波器诊断使用。生产Target必须保持 `PRJ_DRV8870_FACTORY_TEST_ENABLE=0`。

### 2.4 备用TB6612后端

- 每电机使用1路PWM和2路方向GPIO；可选STBY；
- 正确高阻Coast为 `IN1=0、IN2=0、PWM=100%`；
- Brake使用PWM低电平短制动；
- 跨方向命令先进入至少一个上层控制调用周期的Coast，下一周期重复命令后才施加新方向；
- `PRJ_TB6612_STANDBY_CONTROL_ENABLE=0`时，power API只是软件运行闸门，STBY必须由硬件可靠固定为有效；
- 当前 `empty.syscfg` 未包含TB6612方向GPIO，选择TB6612会触发编译门禁；
- `bsp_tb6612_*`属于后端内部接口，应用层禁止直接调用；
- 当前状态仅通过静态设计和构建门禁，尚未完成TB6612硬件验证。

完整切换、引脚历史、验收和回滚见《电机驱动双后端分层与切换指南.md》。
## 3. 编码器

文件：`BSP/Peripherals/bsp_encoder.h`；配置：`project_config.h`的`PRJ_ENCODER_*`、PPR、减速比和方向符号。

- `bsp_encoder_init(cfg,count)`：初始化4路捕获/方向输入。
- `bsp_encoder_get_count(id)` / `get_all_counts(out[4])`：累计计数快照。
- `bsp_encoder_clear(id)` / `clear_all()`：清零；运行控制中慎用。
- `bsp_encoder_get_rpm_mt(id,rpm)` / `get_all_rpm_mt(out[4])`：M/T法测速。
- `bsp_encoder_periodic_update()`：按设计周期更新速度状态，只能由约定任务调用。

机械换算：

```text
counts_per_output_rev = PPR × decode_multiplier × gear_ratio = 13 × 2 × 20 = 520
```

## 4. PID、前馈与辨识

### PID `Application/Algorithm/app_pid.h`

- `app_pid_init(pid,kp,ki,kd,mode,out_min,out_max)`：所有指针有效、输出上下限有序。
- `app_pid_set_params`、`set_integral_limit`、`set_d_filter`、`set_setpoint`：运行时配置。
- `app_pid_compute(pid,feedback,dt_s)`：`dt_s>0`，返回限幅输出。
- `app_pid_reset`：切换状态/启停前复位动态状态。

### 前馈 `app_feedforward.h`

- `app_ff_init`、`app_ff_set_params(k,b)`、`app_ff_set_enabled`。
- `app_ff_compute(ff,target_rpm)`：线性前馈及死区补偿。
- `app_ff_apply_to_pid`：向PID写入前馈。
- `app_ff_fit_linear` / `app_ff_sweep`：实验室标定；Sweep会长时间运行。

### 模型辨识 `app_model_id.h`

- `app_id_start_step(motor,pwm_step)`：`motor=0..3`，幅值1..500。
- `app_id_control_cycle(rpm,out)`：只能由控制周期调用。
- `app_id_abort()`：中止。
- `app_id_process_step`：离线处理采样并输出一阶模型。
- `app_id_pole_placement`：依据带宽和阻尼计算PI。

## 5. 规划与位置控制

- `app_planner_init/start/update/is_done/reset/abort`：梯形速度规划；`dt>0`。
- `app_posctrl_init`：配置位置PID、角度PID、加速度、最大RPM、每转脉冲。
- `app_posctrl_set_mode(ctrl,mode,current_rpm,now_tick)`：模式切换带过渡。
- `app_posctrl_start_position(target_pulses,cruise,current_pos)`。
- `app_posctrl_start_angle(target_angle,cruise,current_yaw)`。
- `app_posctrl_update(...)`：由控制任务周期调用，输出四轮目标。
- `app_posctrl_emergency_stop()`：立即终止规划。

## 6. 串口与协议

### BSP UART `BSP/Peripherals/bsp_uart.h`

- `bsp_uart_init()`；`bsp_uart_putc/getc()`单字节接口。
- `bsp_uart_send(data,len)`阻塞/轮询发送，勿在控制任务使用。
- `bsp_uart_send_dma(data,len)`非阻塞发送；调用前检查`bsp_uart_tx_idle()`；源缓冲区在完成前必须保持有效。
- `bsp_uart_irq_handler()`由板级中断转发调用。

### 协议 `Application/app_vofa.h`

- `app_vofa_parse_cmd(line,cmd)`：解析单行，严格拒绝非有限浮点、越界电机和未知命令。
- `app_vofa_apply_cmd(cmd,ctx,current_motor,need_refresh)`：在菜单任务上下文执行；不要从ISR调用。
- `app_vofa_send_firewater(channels,count)`：文本输出。
- `app_vofa_send_justfloat(channels,count)`：二进制输出。
- `VOFA_TELEMETRY_CHANNEL_COUNT=11`及`vofa_telemetry_channel_t`是协议v1稳定契约。

## 7. ADC、GPIO、Timer、LED

- HAL层`hal_adc/gpio/timer/uart`只负责参数校验和DriverLib转发；BSP层负责项目语义。
- ADC：`bsp_adc_init/read/read_mv`，通道与MEM映射由`project_config.h`和`empty.syscfg`决定；电流值未标定前仅代表放大器输出电压。
- GPIO：`hal_gpio_write_pin/read_pin/read_output_latch`；PB19判断命令状态应读取输出锁存而不是输入电平。
- Timer：`hal_timer_set_pwm_duty`的value必须不超过配置周期。
- LED：`bsp_led_on/off/toggle`，用于低优先级心跳。

## 8. IMU与融合

- `bsp_lsm6dsr_init/update/calibrate/is_stationary`：默认全局上下文简化接口。
- `_ctx`版本允许多实例或测试隔离。
- `app_cf_init/update/get_output/reset`：融合编码器速度、IMU加速度和yaw；`dt_s`必须与实际任务周期一致。

## 9. OSAL

文件：`Lib/OSAL/osal_api.h`。

- 任务：`osal_task_create/delete/delay_ms/get_current`。
- 时间：`osal_get_tick_count`、ms/tick转换。
- 同步：mutex、semaphore、queue及ISR安全版本。
- 临界区：`OSAL_CRITICAL_SECTION`或`osal_critical_enter/exit`；临界区内禁止printf、延时、阻塞UART和复杂计算。

## 10. PC端接口

模块：`tools/mspm0_configurator/src/mspm0_configurator/`。

- `protocol.decode_line(raw)`：区分11列遥测、`@`机器消息和普通文本。
- `CommandBuilder`：对PID、RPM、电机号和运动参数做本地范围校验并生成协议命令。
- `SerialWorker`：QThread串口读写，UI通过signal接收完整行。
- `TelemetryPlot.add_frame/export_csv`：8条RPM/目标曲线和CSV导出。
- `AppSettings.save/load`：PC侧JSON设置；不写MCU Flash。
