# MSPM0G3507 Motor Controller v0.1.0 发布说明

发布日期：**2026-07-18**

## 1. 发布定位

v0.1.0 是当前工程的首个可追溯基线版本。目标是收敛已验证的 DRV8870 电机控制、四路编码器映射、通信协议和配置工具，而不是在发布前进行高风险的大规模任务重写。

## 2. 发布内容

### 固件

- `PROJECT_VERSION_STRING = "0.1.0"`，协议版本 `1`；
- 四路 DRV8870 单 PWM 控制，PB19 高电平使能电机总电源；
- 40%～55%停止死区和 `-500..+500`有符号控制量；
- 四路编码器、速度 PID、前馈、模型辨识、位置/角度控制和 IMU 任务；
- 机器可解析的设备信息、配置和状态响应；
- 11 通道、30 ms 周期的 FireWater 文本遥测。

### PC 配置软件

- Python 3.10+、PySide6、pyserial；
- 中文/英文即时切换；
- 串口扫描和设备查询；
- 四电机 PID、目标速度和前馈设置；
- Run/Stop/StopAll、位置和角度命令；
- 8 条 RPM/目标曲线、11 通道 CSV 保存；
- 原始串口终端和 PC 端 JSON 配置保存。

### 仓库与文档

- 根目录README提供烧录、GUI、协议和安全边界入口；
- `Docs/`按快速使用、架构接口、硬件安全和阶段计划分类；
- `Docs/Plans/`集中保存阶段整改计划和历史审计；
- `.gitignore`排除本地索引、IDE用户状态、构建目录、Python环境和缓存；
- 版本化HEX仅保留在`keil/Release/`。

## 5. 升级与首次运行

1. 保存旧固件和当前 Git 提交；
2. 断开电机机械负载或悬空四轮，使用可限流电源；
3. 烧录生产 HEX；
4. 串口发送 `Info?`，确认 `fw=0.1.0,proto=1`；
5. 发送 `Config?`，核对 PPR、减速比、轮径、轮距和死区；
6. 使用低目标 RPM 分轮验证 A/B/C/D 的方向和编码器符号；
7. 方向、测速和停止行为全部正确后，才允许落地低速闭环测试。

## 6. 回滚

- 固件回滚：重新烧录升级前备份 HEX；
- Git 回滚：切换到升级前提交或标签；
- GUI 回滚：删除本地虚拟环境和工具目录，不影响 MCU；
- 若出现方向异常、无法停止、PB19状态异常或编码器反馈反向，立即使用硬件电源开关断电，不应依赖串口命令完成最后保护。

## 7. 已知风险与不承诺项

- ADC 数据当前只代表放大器输出电压，尚无可靠的安培换算和软件 OCP 门限；
- 软件 PWM 限幅不能替代 DRV8870 芯片保护、保险丝、限流电源和硬件关断；
- 50%中性锁相不等于真实 Coast，硬件连接不变时无法通过软件补足该能力；
- 机械配置是工程默认值，不构成最终标定结果；
- 0.1 GUI 不向 MCU Flash 写入配置，设备重启后仍使用编译期参数；
- UART0 上的人类日志与遥测尚未完全仲裁，密集日志可能影响绘图连续性。
